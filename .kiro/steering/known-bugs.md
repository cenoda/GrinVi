# Known Bugs & Workarounds

## [Kiro IDE] MCP 서버 툴 미사용 시 에이전트 출력이 보이지 않는 버그

**환경:** Kiro IDE (이 워크스페이스)  
**분류:** IDE 환경 버그 (코드 문제 아님)  
**재현 가능 여부:** 항상 재현됨

### 증상

에이전트(Kiro)가 응답을 생성해도 채팅 출력이 화면에 표시되지 않는다.  
MCP 서버 관련 툴을 한 번 호출하면 그 이후부터 출력이 정상적으로 보이기 시작한다.

### 재현 조건

1. Kiro 채팅에서 에이전트에게 메시지를 보낸다.
2. 에이전트가 MCP 서버 툴을 전혀 사용하지 않고 텍스트만 응답하는 경우 → 출력이 안 보임.
3. 에이전트가 MCP 서버 툴(`mcp_shell_exec_run_command`, `mcp_fetch_fetch`, `mcp_openclaw_agent_ask` 등)을 한 번이라도 호출하면 → 이후 출력이 정상적으로 표시됨.

### 원인 추정

Kiro IDE의 스트리밍 렌더링 파이프라인이 MCP 툴 호출 이벤트를 트리거로 초기화되는 것으로 추정.  
MCP 툴 호출 없이 순수 텍스트 응답만 있을 경우 렌더러가 활성화되지 않는 것으로 보임.

### 임시 해결책 (Workaround)

출력이 안 보일 때 에이전트에게 아무 MCP 툴이라도 쓰도록 유도하면 이후 출력이 보임.  
예: "현재 시간 알려줘", "파일 목록 보여줘" 등 툴 호출을 유발하는 질문을 먼저 던진다.
